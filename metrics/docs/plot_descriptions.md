# Phase 1 Plot Descriptions

Generated by `metrics/plot_metrics_demo.py` using a trained agent (checkpoint 198, cmaes_vae_beta2.0, seed 0) rolled out on 16 buffer levels sampled evenly from `buffer_dump_final.npz`.

Plots are saved to `test_log/phase_1/`.

## Figure 1: Pairwise Metric Distributions

**File:** `plots_pairwise_distributions.png`

Histograms of all 120 pairwise metric values (16 levels = 16*15/2 = 120 unique pairs). Four subplots, one per metric:

- **Obs Sequence DTW** -- distribution of normalized DTW distances between flattened observation sequences
- **Position Trace DTW** -- distribution of normalized DTW distances between 2D position traces
- **Value Trajectory DTW** -- distribution of normalized DTW distances between 1D value sequences
- **Spatial Footprint Jaccard** -- distribution of Jaccard indices between visited cell sets

Each histogram has a dashed black line at the mean. Remaining subplots show a summary statistics table (mean, std, min, max for each metric).

**What to look for:** The shape of each distribution tells you about buffer diversity. A tight, unimodal distribution means the buffer levels produce similar agent behavior on that metric. A wide or multimodal distribution indicates genuine diversity. Compare across metrics -- the buffer can be diverse on one axis (e.g., position) but homogeneous on another (e.g., value).

## Figure 2: Pairwise Distance/Similarity Matrices

**File:** `plots_pairwise_matrices.png`

Four 16x16 heatmaps where cell `(i, j)` is the metric value between level `i` and level `j`. The diagonal is rendered in black and excluded from the color scale. Four subplots:

- **Obs Sequence DTW Distance** (colormap: YlOrRd) -- darker = more visually different
- **Position Trace DTW Distance** (colormap: YlOrRd) -- darker = more spatially different paths
- **Value Trajectory DTW Distance** (colormap: YlOrRd) -- darker = more different difficulty profiles
- **Spatial Footprint Jaccard** (colormap: YlGnBu) -- darker blue = higher cell overlap

**What to look for:** Block structure reveals clusters of similar levels in the buffer. Bright rows/columns in distance matrices indicate levels that are close to many others (potential redundancy). Dark rows/columns indicate outlier levels with unique behavior. Cross-referencing matrices shows whether similarity on one metric implies similarity on another.

## Figures 3a-3d: Per-Metric Trajectory Deep-Dives

**Files:**
- `plots_deepdive_pos-dtw-distances.png` -- deep-dive by Position DTW
- `plots_deepdive_obs-dtw-distances.png` -- deep-dive by Observation DTW
- `plots_deepdive_value-dtw-distances.png` -- deep-dive by Value DTW
- `plots_deepdive_jaccard-indices.png` -- deep-dive by Spatial Jaccard

Each figure picks the most similar and most different level pair according to that specific metric. Two rows (similar on top, different on bottom), six columns:

- **Column 1:** Agent's path on level `i`, overlaid on the maze grid. Blue line with time-gradient opacity (faint = early, solid = late). Circle = start, square = end. Walls are dark gray. Title includes step count.
- **Column 2:** Agent's path on level `j` (same format, red line).
- **Column 3:** Position DTW local cost profile (orange) -- per-step alignment cost along the optimal warping path for 2D positions. Low costs = similar locations at that alignment point; spikes = spatial divergence. Y-axis shared across rows.
- **Column 4:** Observation DTW local cost profile (purple) -- per-step alignment cost for flattened 75D observations. Captures visual experience differences beyond position. Y-axis shared across rows.
- **Column 5:** Value DTW local cost profile (seagreen) -- per-step alignment cost for value estimates. Captures difficulty profile divergence. Y-axis shared across rows.
- **Column 6:** Spatial Jaccard overlay -- blue cells visited only by level `i`, red cells only by level `j`, green cells visited by both. Title shows Jaccard index, intersection size, and union size.

**What to look for:** The most similar pair should show comparable trajectory structure across all columns. The most different pair should show divergence. Crucially, each figure selects pairs by a *different* metric, so you can see cases where levels are similar on one metric but different on others. For example, the position DTW deep-dive might show a pair with similar paths but different value curves.

## Figure 4: Spatial Footprints

**File:** `plots_spatial_footprints.png`

Grid of 16 maze visualizations (2 rows x 8 columns), one per level. Each shows:

- **Dark gray cells:** Walls
- **White cells:** Open cells not visited by the agent
- **Blue cells:** Cells visited by the agent during the episode (before first done)
- **Title:** Level index, count of unique cells visited, and episode length (e.g., "L7 (64 cells, 142 steps)")

**What to look for:** The spatial coverage pattern reveals how the agent navigates each level. Trained agents produce coherent paths from start to goal, visible as connected blue corridors. Levels with few visited cells suggest hard mazes where the agent gets stuck. Levels with many visited cells suggest open mazes or mazes requiring extensive exploration. Comparing footprints across levels shows whether the buffer contains structurally diverse mazes or many similar ones.
